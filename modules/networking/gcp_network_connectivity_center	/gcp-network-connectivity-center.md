# Google Network Connectivity Center

## Service overview
Google Network Connectivity Center centralizes hub-and-spoke connectivity management across cloud, branch, and data-center networks.

## When to use Network Connectivity Center
- Many network attachments need centralized governance.
- Hybrid WAN topology spans multiple regions or sites.
- Teams require consistent connectivity onboarding patterns.

## Core capabilities
- Hub-and-spoke topology management model.
- Centralized attachment and route domain governance.
- Scalable orchestration across connectivity types.
- Simplified operational oversight for hybrid networks.

## Real-world usage
- Enterprise WAN modernization programs.
- Multi-region branch and site integration.
- Multi-cloud transit topology governance.

## Operations guidance
- Define hub ownership and change controls.
- Standardize spoke onboarding and validation procedures.
- Continuously audit topology intent versus effective routes.
